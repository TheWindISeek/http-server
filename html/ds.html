Data Structures Overview

This page provides an overview of common data structures used in computer science.

Arrays

An array is a collection of elements stored at contiguous memory locations. Elements can be accessed by their index position.

  • Advantages: Random access, fixed size, easy to implement.
  • Disadvantages: Resizing can be expensive, fixed size.

Linked Lists

A linked list is a sequence of nodes where each node contains a data element and a reference to the next node.

  • Advantages: Easy to insert and delete elements, dynamic size.
  • Disadvantages: No random access, requires more memory due to pointers.

Stacks

A stack is a linear data structure that follows the Last In First Out (LIFO) principle. Elements can only be added or removed from the top.

  • Operations: Push (add element), Pop (remove element).
  • Uses: Function calls, expression evaluation, backtracking.

Queues

A queue is a linear data structure that follows the First In First Out (FIFO) principle. Elements are added at the rear and removed from the front.

  • Operations: Enqueue (add element), Dequeue (remove element).
  • Uses: Serving requests on a single shared resource.

Trees

A tree is a hierarchical data structure consisting of nodes connected by edges. Each node has zero or more child nodes.

  • Types: Binary Trees, Binary Search Trees, AVL Trees, Red-Black Trees.
  • Uses: Organizing hierarchical data, searching, sorting.

Graphs

A graph is a non-linear data structure consisting of nodes (vertices) connected by edges. Graphs can be directed or undirected.

  • Types: Directed Graphs, Undirected Graphs, Weighted Graphs.
  • Uses: Social networks, web pages, transportation networks.
